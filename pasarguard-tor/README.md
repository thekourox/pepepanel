# ابزار خودکارسازی Tor برای پنل Pasargard (Group A Nodes)

این ابزار یک دستیار هوشمند و خودکار برای راه‌اندازی، مدیریت و اتصال گره‌های Tor (که در سیستم با نام Group A شناخته می‌شوند) به پنل مدیریت **پاسارگارد (Pasargard)** است.

## 🏛️ معماری و نحوه کارکرد (بسیار مهم)
**این ابزار باید دقیقاً روی سروری نصب شود که کلاینت‌ها قرار است به آن متصل شوند (سرور Node یا گره).**
به هیچ وجه نیازی نیست که این اسکریپت روی سرورِ اصلیِ پنل پاسارگارد نصب شود.

**روال کار به این شکل است:**
1. شما ابزار را روی سرور Node (مثلاً سرور آلمان) اجرا می‌کنید.
2. ابزار به صورت خودکار نمونه‌های متعددی از Tor را روی پورت‌های لوکال (مثل `127.0.0.1:10001` و ...) در همین سرور اجرا می‌کند.
3. سپس ابزار از طریق پنل وب اختصاصی خود (روی پورت `54322` سرور Node)، با استفاده از API از راه دور به دیتابیس **سرور پاسارگارد** متصل می‌شود.
4. ابزار به هسته (Core) مربوط به این Node در دیتابیس پاسارگارد دستور می‌دهد که کانکشن‌های خروجی (Outbounds) و روتینگ‌ها را به سمت پورت‌های محلی Tor تنظیم کند.
5. در نهایت پاسارگارد کانفیگ جدید را برای هسته Xray در سرور Node می‌فرستد و هسته ری‌استارت می‌شود.

## 🚀 نصب و راه‌اندازی سریع

### پیش‌نیازها
- سیستم‌عامل لینوکس (اوبونتو / دبیان) یا ویندوز.
- پایتون نسخه ۳ به بالا (`python3`).

### دستورات نصب در لینوکس
```bash
git clone <لینک_ریپازیتوری>
cd pasarguard-tor
chmod +x setup_and_run.sh
./setup_and_run.sh
```
پس از اجرای دستور بالا، ابزار تمام پیش‌نیازها را نصب کرده و پنل مدیریت را روی پورت `54322` اجرا می‌کند.
**آدرس دسترسی به پنل:** `http://<IP_SERVER_NODE>:54322`

## 📖 آموزش استفاده از پنل وب (قدم به قدم)

پس از باز کردن پنل تحت وب، مراحل زیر را طی کنید:

### ۱. اتصال به سرور پاسارگارد
1. **Pasargard URL:** لینک پنل اصلی پاسارگارد خود را وارد کنید (مثال: `https://panel.domain.com:54321`).
2. **Pasargard Token:** توکن ادمین پنل پاسارگارد را وارد کنید (Bearer Token).

### ۲. انتخاب هسته (Target Core)
1. روی دکمه آبی‌رنگ **Load** در کنار فیلد Target Core کلیک کنید.
2. ابزار به سرور پاسارگارد وصل شده و لیست تمام هسته‌ها (Cores) را دریافت می‌کند.
3. هسته‌ی مربوط به **همین سرور Node** را از منوی کشویی انتخاب کنید.

### ۳. انتخاب کانفیگ الگو (Template Inbound)
1. روی دکمه **Load Inbounds** کلیک کنید.
2. لیستی از تمام Inbound ها (هاست‌های) موجود روی این هسته نمایش داده می‌شود.
3. یک اینباند را به عنوان الگو (Template) انتخاب کنید. ابزار برای ساخت گره‌های Tor، دقیقاً تنظیمات همین الگو را کپی خواهد کرد.

### ۴. تزریق و ساخت گره‌ها (Inject)
- روی دکمه سبزرنگ **Inject Group A** کلیک کنید.
- ابزار به صورت خودکار به تعداد کشورهای موجود، اینباند جدید می‌سازد و آن‌ها را به دیتابیس پاسارگارد و تنظیمات Xray هسته تزریق می‌کند.

> **نکته امنیتی (OPSEC):** 
> نام‌گذاری گره‌ها کاملاً برای کاربر نهایی تمیز است و هیچ اشاره‌ای به نام Tor نمی‌شود. فرمت اسم‌ها در دیتابیس به این شکل خواهد بود:
> `[A] - Germany - NodeName 🇩🇪`

## ⚙️ مدیریت چرخه حیات (Lifecycle) - فعال و غیرفعال‌سازی

در پایین پنل، سه دکمه قرمز و زرد برای مدیریت گره‌های Group A وجود دارد که به صورت هوشمند و بدون ایجاد اختلال کار می‌کنند:

- **🟡 Disable Group A (غیرفعال‌سازی موقت):**
  اگر می‌خواهید دسترسی کاربران به گره‌های Tor را موقتاً قطع کنید، این دکمه را بزنید. 
  **نحوه کارکرد:** این سیستم علاوه بر خاموش کردن دکمه‌ها در پنل پاسارگارد، **پورت‌های مربوطه را کلاً از داخل فایل کانفیگ Xray حذف کرده و هسته را ری‌استارت می‌کند.** بنابراین در زمان Disable بودن، گرفتن Ping از پورت‌ها کاملاً غیرممکن خواهد بود.

- **🟢 Enable Group A (فعال‌سازی مجدد):**
  این دکمه گره‌هایی که غیرفعال کرده بودید را دوباره بیدار می‌کند. پورت‌ها را دقیقاً با همان آیدی و تگ‌های قبلی مجدداً وارد کانفیگ Xray کرده و هسته را ری‌استارت می‌کند.

- **🔴 Cleanup Group A (حذف کامل):**
  این دکمه تمام گره‌های Tor ساخته شده، خروجی‌ها (Outbounds) و روتینگ‌های مربوط به آن‌ها را به طور کامل و برای همیشه از دیتابیس پاسارگارد و هسته Xray پاک می‌کند.

## 🛠 رفع اشکال (Troubleshooting)
- **خطای 404 در زمان Load:** مطمئن شوید که آدرس پنل پاسارگارد را با پورت صحیح وارد کرده‌اید.
- **ارور Unprocessable Entity (422):** مطمئن شوید که تمام فیلدها (Core ID و Template ID) را به درستی Load و انتخاب کرده‌اید.
- **گره‌ها در لیست پنل ساخته شده‌اند اما اینترنت ندارند:** مطمئن شوید که روی سرور Node محدودیت فایروال محلی روی پورت‌های Tor (بازه 10001 به بالا) وجود ندارد و پروسه‌های Tor در بک‌گراند سیستم به درستی ران شده‌اند.
